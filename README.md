## 1. Функциональность программы
Игра «жизнь» это  клеточный автомат который создает случайным образом начальную популяцию сообщества клеток, а затем руководствуясь рядом правил, создает следующее поколение по следующим правилам:
* Место действия игры это размеченная на клетки замкнутая поверхность.
* Каждая клетка на этой поверхности может находиться в двух состояниях: быть «живой» (заполненной) или быть «мёртвой» (пустой). Клетка имеет восемь соседей, окружающих её.

Каждое следующее поколение рассчитывается на основе предыдущего по таким правилам:
* в пустой (мёртвой) клетке, рядом с которой ровно три живые клетки, зарождается жизнь;
* если у живой клетки есть две или три живые соседки, то эта клетка продолжает жить; в противном случае, если соседей меньше двух или больше трёх, клетка умирает («от одиночества» или «от перенаселённости»)


Игра прекращается, если
* на поле не останется ни одной «живой» клетки
* конфигурация на очередном шаге в точности (без сдвигов и поворотов) повторит себя же на одном из более ранних шагов (складывается периодическая конфигурация)
* при очередном шаге ни одна из клеток не меняет своего состояния (складывается стабильная конфигурация; предыдущее правило, вырожденное до одного шага назад)
## 2. Установка программы
склонировать проект из репощитория:

`$ git clone https://github.com/trpo2021/cw-ip-017-game-life.git`
 
 Перейдём в папку с проектом:
 
`$ cd cw-ip-017-game-life`

Соберём приложение с помощью gradle:

`$ ./gradlew build`

Перейдём в папку с собранным приложением:

`$ cd build/libs/`

Запустим приложение командой

`$ java -jar GameOfLife-0.1.jar --height 7 --speed 100`

## 3. Формат входных данных:
Пользователь может указать начальное кол-во живых клеток и скорость появления новых поколений.

**Ключи аргументов командной строки:**

```--width int``` - ширина игрового поля целое число от нуля (по умолчанию 100)

```--height int``` - высота игрового поля целое число от нуля (по умолчанию 30)

```--speed int``` - скорость появления новых поколений, целое число от 100 до 5000 миллисекунд (по умолчанию 500 миллисекунд).

```--alivePercent double ``` - начальный процент живых клеток, рациональное неотрицательное число от 0 до 100% (по умолчанию 30% живых клеток).

Данные ключи не являются обязательными и при их отсутствии будут использованы значения по умолчанию.
